== Changelog
V2.0.16
[fixes]
- Watchdog timer reworked: idle time is now anchored to wall-clock (UTC) instead of per-minute increments, supports fractional minutes (0.25 step), no longer chops an in-progress transmission (disable is deferred to a TX-off boundary), and starts a fresh window on a new DX selection or a reset click. The WD label now shows seconds resolution (e.g. WD:45s, WD:1m30s) (PR #79, SpudGunMan).
- Auto-QRM auto-stop hardened: the stop now triggers only when the selected DX station is in a directed exchange with someone other than you, using base-callsign normalization for sender/target matching. Reduces false stops and missed stops (PR #79, SpudGunMan).
- FT2 band hopping fixed: toggleBands now maps FT2 to its own band schedule instead of folding it into FT4; band-hop boundary moved to the AutoCall->AutoCQ transition (PR #79, SpudGunMan).
- Ignore-list adds are now de-duplicated (base-callsign aware) across the manual button, the right-click action, and watchdog-timeout auto-ignore (PR #79, SpudGunMan).
- Erase button now routes through the canonical Stop TX path (stopTxButton->click()) for both right-click-erase and triple-erase, replacing duplicated manual tx-stop sequencing (PR #85, SpudGunMan).

[features]
- New "Mode Now" and "Band Now" buttons immediately toggle AutoCQ<->AutoCall or hop bands without waiting for the auto cycle; shown only when the corresponding auto feature is enabled (PR #84, SpudGunMan).

[tooling]
- Fix build with modern GCC/gfortran 14+: rename Fortran local SPLIT subroutine to JPLSPLIT (avoids clash with gfortran 14's new F2023 SPLIT intrinsic) and suppress a spurious -Wmaybe-uninitialized around placement-new HRDMessage usage. Tested on Debian, macOS, and JTSDK4111 (PR #77, SpudGunMan; issue #75).

V2.0.15
[features]
- Pile-up mode for Auto Call: right-click the Auto Call checkbox to toggle. When enabled, all continent filters are temporarily disabled so you respond to stations calling you directly without adding to the pile-up. Continent settings are saved and restored when pile-up mode is toggled off. The Auto Call label turns red while active (PR #73, SpudGunMan).
- PSK Reporter band-activity feedback: when PSK Reporter spotting is enabled, band-activity lines from stations also heard by PSK Reporter receivers on the current band+mode are visually highlighted (bold or underline) so you can see at a glance which decodes are being copied elsewhere (PR #69, SpudGunMan).

[tooling]
- WSJT-Z Debug mode now emits significantly more diagnostic context: startup config dump (FT8 thread count, decoder start mode, sensitivity, cycles, etc.), platform info (OS, arch, kernel), and per-branch state in auto_sequence / processMessage / on_autoButton_clicked. Helps triage support reports without asking the operator to reproduce in custom debug builds (PR #68, SpudGunMan).
- New UDP example: `UDPExamples/decode_WSJT-UDP.py`, a Python reference for parsing the WSJT-X UDP decode stream (PR #72, SpudGunMan).

V2.0.13
[fixes]
- Auto-sequence stop condition now requires the selected DX call to actually be the transmitting station in the decode, and the reply target to not be your own call. Previously the stop could trigger when the selected DX was being called by someone else (PR #25, SpudGunMan).
- Watchdog idle timer now correctly pauses while Auto-Call is active (in addition to Auto-CQ). Previously Auto-Call sessions could trip the watchdog and halt TX even though the operator was actively cycling (PR #24, SpudGunMan).
- Tailender detection now treats RR73 and RRR as end-of-QSO acknowledgements alongside 73, so Auto-CQ/Auto-Call no longer reply to a station that has already signed off (PR #24, SpudGunMan).
- Band-hopper auto-mode logic hardened against edge cases when band switches coincide with mode changes (related to issue #22) (PR #26, SpudGunMan).

[features]
- New status labels for Band Hop ("BH") and Mode Switch ("MS") show the countdown to the next scheduled change and the next band/mode, so the operator can see at a glance what the auto-cycle is about to do (PR #27, SpudGunMan).

[tooling]
- macOS / Apple ARM cross-compile support — the project now builds on Apple Silicon, Ubuntu (ARM/Intel), and Pi 5. New `doc/build-osx.sh` script bootstraps the macOS SDK for cross-compilation, tested against v26-OSX (PR #23, SpudGunMan).
- Version numbering consolidated to a single source of truth: `set (VERSION_Z "X.Y.Z")` in CMakeLists.txt now drives both the package filename and the in-app title bar (via `wsjtx_config.h.in`). Bumping the WSJT-Z mod version is now a one-line edit (PR #21, SpudGunMan).

V2.0.12
[fixes]
- Echo mode no longer segfaults on first decode. The C++ side was passing a struct whose layout did not match the Fortran params_block expected by the Echo decoder, corrupting memory as soon as the Echo path ran. Layouts are now in sync (PR #16, SpudGunMan).
- PSKReporter widget no longer triggers Qt thread-safety warnings (and the occasional crash that came with them). The reception-report table was being populated from a QtConcurrent worker thread, which is illegal for QWidget objects; population now runs on the GUI thread (PR #17, SpudGunMan).
- Auto-CQ no longer fires a stray CQ when not in Auto-CQ mode. A code path in the Auto-Call flow could trigger a CQ transmission unintentionally (PR #19, SpudGunMan).
- Auto-CQ free-slot checker no longer reshuffles your TX frequency every cycle for no reason. If your current TX slot is still free it is kept; otherwise the search starts from the current frequency and walks outward in alternating ±10 Hz steps with band-edge wrap-around, instead of always re-anchoring at 1500 Hz (PR #18, SpudGunMan).

[performance]
- Loaded WAV files are now decoded across the full 200–4000 Hz range for FT8/FT4/FT2, matching jt9 defaults. The WideGraph window is bypassed for disk playback, so decodes outside the live receive window are no longer dropped when re-analyzing recordings (PR #15, SpudGunMan).
- The FT8 multi-thread duplicate filter is now gated by the "Hide FT8 dupe messages" menu setting, and is always on for disk playback. Operators who want raw decoder output can untick the menu item to disable dedup on live air (PR #15, SpudGunMan).

[tooling]
- Configuration UI: cleaned up duplicate widget object names. No user-visible change; reduces ambiguity for future findChild lookups and keeps Qt Designer happier (PR #17, SpudGunMan).

V2.0.11
[fixes]
- TCI audio now actually starts. The "Use TCI Audio" checkbox was wired into Settings but the resulting flag bit was never OR'd into rig_params.poll_interval, so TCITransceiver booted with tci_audio_=false and never sent audio_start to the rig. Result: no RX audio reached WSJT-Z, and TX audio only worked if the operator had manually selected TCI as the audio source on the rig side. Configuration::gather_rig_data now sets the tci__audio bit when the checkbox is ticked, matching the upstream wiring.

V2.0.10
[fixes]
- Contest log save no longer fails with "Database Error: No query / Unable to fetch row" after decoding any US callsign. Root cause: the USState lookup was calling QSqlDatabase::addDatabase("QSQLITE") with no connection name, which replaces the unnamed default SQLite connection used by the contest log. Fix: USState now uses its own named connection ("usstate") opened once and reused.
- App no longer crashes on Settings → OK when "Special operating activity" is enabled and the Advanced tab has been visited. Same root cause as above: the settings flow calls on_contest_log_action_triggered → n_qso(), the SQL error threw std::runtime_error out of a Qt slot, and Qt aborts the process on uncaught exceptions. n_qso() now returns 0 on a SELECT failure instead of throwing.

V2.0.7
[fixes]
- Fixed FT8 multi-threaded decoder producing duplicate decodes of the same station on the same frequency within a single cycle (e.g. same message with SNR +1 and -18)
- ALLCALL7.TXT now reliably found by jt9 regardless of how wsjtx is launched (jt9 working directory set to install bin/ folder)
- Fixed FT8 MTD duplicate decodes: the earlyDecodes dedup buffer was being cleared at the top of decode(), which runs twice per cycle in multi-threaded mode — wiping the buffer between the early-pass and final-pass, so dedup never worked. Reset moved to a period boundary in guiUpdate() (seconds 10/25/40/55, ~2s before the next cycle's early decode), matching wsjtx-improved
- jt9 no longer survives a wsjtx crash on Windows: proc_jt9 is now assigned to a Job Object with JOB_OBJECT_LIMIT_KILL_ON_JOB_CLOSE, so when wsjtx exits for any reason (clean, crash, Task Manager kill) Windows terminates jt9 automatically

[performance]
- Cached hot-path regexes in displaytext.cpp (grid_regexp, AP-info stripper, AP-position locator) as file-scope statics — previously recompiled on every decoded line
- Cached " 30[ABCD] " regex in activeStations.cpp display loop
- Batched ActiveStations redraws via a 300ms single-shot QTimer: addLine() now only updates the internal map; the full QTextEdit rebuild+repaint happens at most once per tick (previously every decoded line), fixing UI stalls on crowded FT8 bands. Also fixed a latent bug where addLine wrote to a local variable shadowing the member
- read_log() (Q65 startup scan of wsjtx.log) moved to QtConcurrent so the main window paints immediately; results merged on the UI thread via QFutureWatcher
